# HanaBi 開発ルール

## プロジェクト概要

Fire TV向け NAS動画プレイヤーアプリ（KODIの代替）。詳細は `README.md` 参照。

- **アーキテクチャ**: MVVM + Clean Architecture
- **動画構造**: `アニメ > 作品名 > シーズン > エピソード.mp4`

## 開発環境

- **Windowsパス（正）**: `I:\dev\HanaBi`
- **WSLパス**: `/mnt/i/dev/HanaBi`
- ビルド・実行は Android Studio で行う（WSLパスからは開けないバグあり）
- Claude は WSL 側から起動する

## 開発フロー

1. 不明点は `AskUserQuestion` で確認
2. 変更対象ファイル・方針をユーザーに提示して承認を得る
3. 計画に沿ってコードを変更する
4. ユーザーによる動作確認（Androidビルドのためユーザーが実機確認）
5. ユーザーの確認後にのみ `git commit` する

## Planモード

- 3ステップ以上 or アーキテクチャに関わるタスクは必ず Plan モードで開始する
- 途中でうまくいかなくなったら、無理に進めずすぐに立ち止まって再計画する

## タスク管理

- 計画は `tasks/todo.md` にチェックリスト形式で記録する
- 修正を受けたら `tasks/lessons.md` にパターンを記録する（同じミスを繰り返さない）

## コア原則

- **シンプル第一**: 影響するコードを最小限にする。過剰設計しない
- **根本原因を修正する**: 一時的な修正は避ける
- **動作証明**: 動作を証明できるまでタスクを完了とマークしない

## Git 運用ルール

- ブランチ: `main`（本番）, `feature/xxx`（機能開発）
- コミット・プッシュはユーザーの確認後のみ（自動実行禁止）
- `.env` / `local.properties` は絶対にコミットしない

## 禁止事項

- ユーザーの確認なしに `git commit` / `git push` を実行しない
- `local.properties` を編集・コミットしない（NAS認証情報等が含まれる可能性）
- Android ビルドを自動実行しない（Claude からは実行できない）
